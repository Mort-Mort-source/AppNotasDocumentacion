\section{Introducción}
Smart Notes es una aplicación móvil desarrollada en Flutter que permite gestionar notas personales organizadas en grupos (carpetas). El proyecto implementa funcionalidades de arrastrar y soltar, personalización de colores, historial de edición (Undo/Redo) y persistencia local mediante Hive.

La característica principal es su \textbf{asistente IA integrado}, que analiza automáticamente el contenido de las notas y sugiere acciones útiles como configurar recordatorios automáticos (alarmas) y crear o mover notas a grupos temáticos.

La aplicación cuenta con un diseño minimalista, sistema de grupos, arrastrar y soltar, y notificaciones locales.

\section{Tecnologías utilizadas}
\begin{itemize}
    \item \textbf{Lenguaje:} Dart 3.x
    \item \textbf{Framework:} Flutter 3.x
    \item \textbf{Persistencia local:} Hive + Hive Flutter (NoSQL, rápido y nativo)
    \item \textbf{Generación de código:} build\_runner, hive\_generator
    \item \textbf{Notificaciones:} flutter\_local\_notifications
    \item \textbf{Inteligencia Artificial:} OpenRouter
    \item \textbf{IDE:} Visual Studio Code
    \item \textbf{Control de versiones:} Git
\end{itemize}

\section{Integración con OpenRouter (IA)}

Para dotar a la aplicación de inteligencia, utilizamos la API de \textbf{OpenRouter}, un servicio que permite acceder a múltiples modelos de lenguaje avanzados.

\subsection{Cómo funciona la integración:}
\begin{enumerate}
    \item Cuando el usuario guarda una nota, se llama automáticamente al servicio \texttt{AIService}.
    \item Se envía el título y contenido de la nota junto con la fecha actual al modelo de IA.
    \item La IA analiza el texto en lenguaje natural y responde en formato JSON estructurado.
    \item El JSON contiene:
    \begin{itemize}
        \item \texttt{has\_date}: Si detecta una fecha u hora
        \item \texttt{date\_time}: Fecha y hora en formato YYYY-MM-DD HH:MM
        \item \texttt{category}: Categoría de la nota
        \item \texttt{suggested\_action}: ``alarma'', ``crear\_grupo'' o ``ninguna''
        \item \texttt{explanation}: Explicación breve para el usuario
    \end{itemize}
    \item Según la sugerencia, la app muestra un diálogo elegante ofreciendo al usuario aceptar o ignorar la recomendación.
\end{enumerate}

Esta integración permite que la aplicación entienda el contexto de las notas y ayude proactivamente al usuario.

\section{Configuración Manual de Alarmas}

Además de las sugerencias automáticas de la IA, la aplicación permite configurar recordatorios de forma manual:

\begin{itemize}
    \item Al crear o editar una nota, el usuario puede presionar el icono de alarma.
    \item Se abre un selector de fecha y hora.
    \item Una vez guardada la nota, se programa una notificación local mediante \texttt{flutter\_local\_notifications}.
    \item El usuario puede eliminar o modificar el recordatorio en cualquier momento.
\end{itemize}

\section{Heurísticas de Nielsen: Evaluación del Proyecto}

A continuación se analiza cómo cumple la aplicación con las 10 heurísticas de usabilidad de Jakob Nielsen:

\begin{enumerate}
    \item \textbf{Visibilidad del estado del sistema:} Cumple. La aplicación muestra siempre el estado actual mediante \texttt{ValueListenableBuilder}.
    \item \textbf{Relación entre el sistema y el mundo real:} Cumple. Usa lenguaje cotidiano e iconos reconocibles.
    \item \textbf{Control y libertad del usuario:} Cumple. Permite deshacer/rehacer, cancelar diálogos y navegar hacia atrás.
    \item \textbf{Coherencia y estándares:} Cumple. Sigue las convenciones de Material Design.
    \item \textbf{Prevención de errores:} Cumple parcialmente. Valida título vacío y pide confirmación antes de eliminar.
    \item \textbf{Reconocimiento en lugar de recuerdo:} Cumple. Todas las acciones están visibles.
    \item \textbf{Flexibilidad y eficiencia de uso:} Cumple. Permite arrastrar y soltar para usuarios avanzados.
    \item \textbf{Diseño estético y minimalista:} Cumple. Interfaz limpia con colores pastel.
    \item \textbf{Ayuda a los usuarios a diagnosticar errores:} Cumple parcialmente.
    \item \textbf{Ayuda y documentación:} No cumple. Falta una pantalla de ayuda o tutorial.
\end{enumerate}

\section{Metas Cumplidas y Comparación}

Se cumplieron las siguientes metas principales:
\begin{itemize}
    \item Sistema completo de notas con persistencia local usando Hive.
    \item Gestión avanzada de grupos (creación, edición, eliminación y asignación de notas).
    \item Interacciones táctiles avanzadas: arrastrar y soltar (nota sobre nota, nota sobre grupo, eliminación por arrastre).
    \item Personalización visual (6 colores pastel).
    \item Historial de edición (Undo/Redo).
    \item Integración exitosa con OpenRouter para sugerencias inteligentes de alarmas y grupos.
    \item Notificaciones locales programables.
\end{itemize}

Las funcionalidades de arrastrar y soltar y la integración con IA superaron las expectativas iniciales.

\section{Áreas que Quedaron Pendientes y Mejoras Futuras}

Aunque se avanzó significativamente, quedaron pendientes las siguientes funcionalidades:
\begin{itemize}
    \item Reconocimiento de texto mediante OCR (foto → nota).
    \item Reconocimiento de voz (dictado).
    \item Búsqueda global de notas.
    \item Manejo robusto de errores de red y límites de la IA.
    \item Sincronización en la nube.
\end{itemize}

\section{Conclusión y Aprendizaje}

El desarrollo de \textbf{Smart Notes} ha sido una experiencia muy enriquecedora. Logré integrar tecnologías modernas como Flutter, Hive y OpenRouter, comprendiendo profundamente el ciclo completo de desarrollo de una aplicación móvil: desde el diseño de la arquitectura, la persistencia de datos, las interacciones táctiles avanzadas, hasta la integración de inteligencia artificial.

Aprendí la importancia de una buena experiencia de usuario, la utilidad de las heurísticas de Nielsen y cómo la IA puede transformar una aplicación sencilla en una herramienta realmente inteligente y útil.

Este proyecto me permitió consolidar conocimientos de programación móvil, diseño de interfaces y desarrollo de funcionalidades complejas, preparándome mejor para futuros desafíos en el campo de la Interacción Humano-Computadora.
