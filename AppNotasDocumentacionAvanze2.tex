\section{Introducción}
Smart Notes es una aplicación móvil desarrollada en Flutter que permite crear, organizar y gestionar notas de forma inteligente. 

La característica principal es su \textbf{asistente IA integrado}, que analiza automáticamente el contenido de las notas y sugiere acciones útiles como:
\begin{itemize}
    \item Configurar recordatorios automáticos (alarmas)
    \item Crear o mover notas a grupos temáticos
\end{itemize}

La aplicación cuenta con un diseño minimalista, sistema de grupos, arrastrar y soltar, y notificaciones locales.

\section{Tecnologías Utilizadas}
\begin{itemize}
    \item \textbf{Flutter} + Dart (Framework multiplataforma)
    \item \textbf{Hive} (Base de datos NoSQL ligera y rápida)
    \item \textbf{flutter\_local\_notifications} (Para recordatorios)
    \item \textbf{OpenRouter} (Servicio de Inteligencia Artificial)
    \item Material Design 3 (Interfaz moderna y limpia)
\end{itemize}

\section{Integración con OpenRouter (IA)}

Para dotar a la aplicación de inteligencia, utilizamos la API de \textbf{OpenRouter}, un servicio que permite acceder a múltiples modelos de lenguaje avanzados.

\subsection{Cómo funciona la integración:}
\begin{enumerate}
    \item Cuando el usuario guarda una nota, se llama automáticamente al servicio \texttt{AIService}.
    \item Se envía el título y contenido de la nota junto con la fecha actual al modelo de IA.
    \item La IA analiza el texto en lenguaje natural y responde en formato JSON estructurado.
    \item El JSON contiene:
    \begin{itemize}
        \item \texttt{has\_date}: Si detecta una fecha u hora
        \item \texttt{date\_time}: Fecha y hora en formato YYYY-MM-DD HH:MM
        \item \texttt{category}: Categoría de la nota
        \item \texttt{suggested\_action}: ``alarma'', ``crear\_grupo'' o ``ninguna''
        \item \texttt{explanation}: Explicación breve para el usuario
    \end{itemize}
    \item Según la sugerencia, la app muestra un diálogo elegante ofreciendo al usuario aceptar o ignorar la recomendación.
\end{enumerate}

Esta integración permite que la aplicación entienda el contexto de las notas y ayude proactivamente al usuario.

\section{Configuración Manual de Alarmas}

Además de las sugerencias automáticas de la IA, la aplicación permite configurar recordatorios de forma manual:

\begin{itemize}
    \item Al crear o editar una nota, el usuario puede presionar el icono de alarma.
    \item Se abre un selector de fecha y hora.
    \item Una vez guardada la nota, se programa una notificación local mediante \texttt{flutter\_local\_notifications}.
    \item El usuario puede eliminar o modificar el recordatorio en cualquier momento.
\end{itemize}

Esta funcionalidad asegura que el usuario siempre tenga control total sobre sus recordatorios, incluso si decide no usar las sugerencias de la IA.

\section{Heurísticas de Nielsen: Evaluación del Proyecto}
A continuación se analiza cómo cumple la aplicación con las 10 heurísticas de usabilidad de Jakob Nielsen:

\begin{enumerate}
    \item \textbf{Visibilidad del estado del sistema:} \textit{Cumple}. La aplicación muestra siempre el estado actual (título en AppBar, lista de notas actualizada en tiempo real mediante \texttt{ValueListenableBuilder}).
    \item \textbf{Relación entre el sistema y el mundo real:} \textit{Cumple}. Usa lenguaje cotidiano (``Nota'', ``Grupo'') e iconos reconocibles.
    \item \textbf{Control y libertad del usuario:} \textit{Cumple}. Permite deshacer/rehacer, cancelar diálogos y navegar hacia atrás.
    \item \textbf{Coherencia y estándares:} \textit{Cumple}. Sigue las convenciones de Material Design.
    \item \textbf{Prevención de errores:} \textit{Cumple parcialmente}. Valida título vacío y pide confirmación antes de eliminar. Falta validar grupos duplicados.
    \item \textbf{Reconocimiento en lugar de recuerdo:} \textit{Cumple}. Todas las acciones están visibles (menús, iconos, botones).
    \item \textbf{Flexibilidad y eficiencia de uso:} \textit{Cumple}. Permite arrastrar y soltar para usuarios avanzados, además de menús contextuales.
    \item \textbf{Diseño estético y minimalista:} \textit{Cumple}. Interfaz limpia, colores pastel, sin elementos superfluos.
    \item \textbf{Ayuda a los usuarios a diagnosticar errores:} \textit{Cumple parcialmente}. Muestra mensajes claros al fallar la validación, pero no maneja errores de red de la IA.
    \item \textbf{Ayuda y documentación:} \textit{No cumple}. No existe una pantalla de ayuda o tutorial integrado.
\end{enumerate}

En general, el proyecto cumple con la mayoría de las heurísticas, destacando la usabilidad y la eficiencia en las tareas principales.

\section{Áreas de Mejora y Retos Futuros}
A pesar del buen funcionamiento actual, se identifican áreas de mejora:

\begin{itemize}
    \item \textbf{Límite de tokens en modelos gratuitos:} Los modelos gratuitos de OpenRouter tienen un límite de tokens por minuto (30 req/min, 60k tokens/min). Esto puede ser insuficiente en producción si la aplicación escala. Como alternativa se podría implementar caché local de respuestas o migrar a un modelo de pago con mayor capacidad.
    \item \textbf{Manejo de errores de red:} Actualmente, si la IA falla por falta de conexión, la app solo muestra un mensaje genérico. Se podría agregar un reintento automático o un modo offline con sugerencias locales.
    \item \textbf{Sincronización en la nube:} La aplicación guarda solo en local. Un futuro desarrollo podría incluir sincronización con servicios en la nube para respaldo y uso multiplataforma.
    \item \textbf{Ampliación de la detección de entidades:} Mejorar el prompt de IA para reconocer horarios más complejos (ej. ``cada lunes'', ``dentro de dos semanas'') y sugerir recordatorios recurrentes.
    \item \textbf{Privacidad de los datos:} Asegurar que la API key esté protegida y que los datos de usuario no se compartan con terceros sin consentimiento.
\end{itemize}

\section{Conclusiones}
Smart Notes combina una interfaz moderna y fácil de usar con un asistente inteligente basado en OpenRouter, logrando una experiencia de usuario avanzada y útil. 

La aplicación demuestra cómo la inteligencia artificial puede ser integrada de forma práctica en herramientas cotidianas para mejorar la productividad, cumpliendo con los principios fundamentales de interacción humano-computadora.
